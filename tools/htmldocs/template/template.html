    ABCD TEST;option=something Hello World

    "Hello World" always is a good start

  • Server
    +ACK ABCD 123 Hello user
  • Server
    +ABCD 125 I can send more lines

    The server can send more lines. Really!

  • …
  • Server
    +ABCD 4200 Hello, are you still there? Terminate me if you can!

    Oh no, the server wonders if we're still there! We are even asked to terminate the command... Maybe we should do this? (after this overly long remark which is just there to check how an overflow will look like)

  • Client
    ABCD TERM

  • Client has requested termination, server will close the channel soon...

  • Server
    -ACK ABCD 4205

Chapter 4

DRCI

Claim DataRef Immediate

DataRef Claim Immediate
Command Version 1

Attempts to create the specified dataref within X-Plane. Claimed datarefs need to be released through channel termination before redefinition; definitions cannot be shared between channels or sessions.

If the dataref is manipulated, accepted (optionally immediately adjusted) input can be retrieved from the dataref again. The updated value is sent to the channel asynchronously.

"Immediate" means two things:

  • value changes are direct, not adjusted over time (other than input to a PID controller)
  • value changes are not held back for external confirmation

The dataref can be accessed from X-Plane as well as via all other XPRC commands (e.g. DRQV and DRMU, it is also seen by DRLS).

A single registration can be used for multiple types, as long as a compatible conversion method exists.

Options:
echo=X

controls if and whose values should be forwarded to the channel when set

Possible value suffixes:
ms

milliseconds
f

frames
Constants for X:
all

notify about all values being set, regardless of source
other

notify only about values received through X-Plane or XPRC sessions other than the current one
xplane

notify only about values received through X-Plane (updates from XPRC are not forwarded)
none

disables all notifications
repeatFreq=X

repeat in intervals of X

Suffixes for X:
ms

milliseconds
f

frames

    Condition: only if we feel like it

    value type

    1. this is a sub-parameter, reusing same template as for main level

  2. name of the requested dataref

  3. Multiplicity: optional
    Condition: optionally, if array or blob and I'm writing even more here to check how it would look like if we wrap

    [i] to set only values from index i (i=0 is the first element; default=0)

    This is a second paragraph.

Response:

  1. status

    • If dataref cannot be claimed or types are different than already claimed: ERR
    • Otherwise: ACK

  2. When the dataref value has been touched, sent according to echo option:

    • value(s) at referenced time
      • see dataref type descriptions for value encoding
      • if the dataref has been registered for multiple types, values are separated by semicolon
        • in order specified as # in type overview (note: not matching order from command parameter)

  3. Something else.

    Just for the template. Not actually sent.
